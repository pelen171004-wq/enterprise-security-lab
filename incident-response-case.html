Case Study: Guest VLAN Attempted Lateral Movement

This incident case documents a simulated security event where an untrusted guest endpoint attempted to reach protected internal assets. The event was detected, blocked and reviewed as part of an incident response workflow.

Incident Summary

  • Incident: Guest VLAN attempted access to Users VLAN
  • Source Zone: VLAN60 – Guest
  • Target Zone: VLAN20 – Users
  • Detection Method: Firewall deny policy + SOC event monitoring
  • Impact: No access granted, traffic blocked at policy layer
  • Status: Contained and documented

Root Cause & Containment

  • Root Cause: Untrusted endpoint initiated unauthorized east-west traffic.
  • Containment: Firewall deny rule prevented lateral movement.
  • Validation: Policy map and telemetry confirmed no successful session.
  • Follow-up: Event recorded for incident review and tuning validation.
