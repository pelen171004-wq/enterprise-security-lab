Troubleshooting Workflow

Praktický debugging flow pre FortiGate, CBS250 a VLAN segmentáciu.

1. Troubleshooting flow

L1 / link up? ↓ VLAN correct? ↓ Client IP / mask / gateway correct? ↓ Can client ping gateway? ↓ Is route present on firewall? ↓ Does firewall policy match? ↓ Is NAT enabled if going to WAN? ↓ DNS works? ↓ Debug flow / sniffer if still broken

2. FortiGate commands

show system interface show firewall policy show full-configuration firewall local-in-policy get router info routing-table all execute ping 8.8.8.8 diagnose firewall iprope lookup ... diagnose debug flow ... diagnose sniffer packet any 'host 1.1.1.1' 4

3. CBS250 commands

show vlan show interfaces status show interfaces switchport show running-config show mac address-table show spanning-tree interface giX switchport mode access switchport access vlan 20 switchport mode trunk

4. Typical problem patterns

  • Klient je v zlej VLAN
  • Chýba default gateway
  • FortiGate policy nepasuje na src/dst intf
  • Chýba NAT smerom na WAN
  • DNS funguje zle, ale routing je v poriadku
  • Management access blokuje local-in policy alebo trusted host
