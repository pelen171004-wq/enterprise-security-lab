Troubleshooting

FortiGate 60E and CBS250 diagnostic commands for common failure scenarios. Includes packet flow debugging, VLAN verification, and policy troubleshooting.

FortiGate 60E — Diagnostics

FORTIGATE CLI
Packet Flow Debug — trace traffic path through FW
# Step 1: Reset and set filter
FG60E # diagnose debug reset
FG60E # diagnose debug flow filter addr 10.10.20.50
FG60E # diagnose debug flow show function-name enable
FG60E # diagnose debug flow show iprope enable
# Step 2: Start trace
FG60E # diagnose debug flow trace start 100
FG60E # diagnose debug enable
# Step 3: Reproduce the issue, then stop
FG60E # diagnose debug disable
FG60E # diagnose debug reset
Packet Sniffer — capture traffic on interface
# Sniff on VLAN20 — host filter
FG60E # diagnose sniffer packet vlan20 'host 10.10.20.50' 4
# Sniff all interfaces — port 53 DNS
FG60E # diagnose sniffer packet any 'port 53' 4 0 l
# Sniff WAN inbound
FG60E # diagnose sniffer packet wan1 'tcp and port 443' 6
# Stop sniffer
# Press Ctrl+C
Session & Policy Verification
# List active sessions from VLAN20
FG60E # diagnose sys session filter src 10.10.20.0/24
FG60E # diagnose sys session list
# Check policy hit counters
FG60E # diagnose firewall iprope show 100004 0
# Verify routing table
FG60E # get router info routing-table all
# Check ARP table
FG60E # diagnose ip arp list
# Check DHCP leases on VLAN20
FG60E # diagnose sys dhcp server list
IPS & UTM Status
# IPS engine status
FG60E # get ips status
# IPS anomaly detections
FG60E # diagnose ips anomaly list
# FortiGuard update status
FG60E # diagnose autoupdate status
# Web filter category test
FG60E # diagnose test application urlfilter 3
# SSL inspection certificate check
FG60E # diagnose debug application ssl -1

CBS250 — Diagnostics

CBS CLI
VLAN & Port Verification
# Show all VLANs
SW-A# show vlan
# Show specific VLAN
SW-A# show vlan id 20
# Show interface status all ports
SW-A# show interfaces status
# Show trunk port config
SW-A# show interfaces gi1 trunk
# Show MAC address table for VLAN20
SW-A# show mac address-table vlan 20
# Show running config
SW-A# show running-config
Common Failure — Host not getting IP on VLAN20
# 1. Verify port is in correct VLAN
SW-A# show interfaces gi5
  Access Mode VLAN: 20 (USERS)
  Operational Mode: Access

# 2. Verify VLAN20 exists on trunk to FortiGate
SW-A# show interfaces gi1 trunk
  VLANs allowed and active in management domain: 20,30,60

# 3. Check DHCP on FortiGate
FG60E # diagnose sys dhcp server list
# 4. Check FortiGate interface vlan20 is up
FG60E # diagnose hardware deviceinfo nic vlan20
Common Failure — Guest VLAN60 can reach Users VLAN20
# 1. Verify deny policy exists on FortiGate
FG60E # show firewall policy
# Look for: srcintf=vlan60 dstintf=vlan20 action=deny

# 2. Debug flow from guest host
FG60E # diagnose debug flow filter src 10.10.60.100
FG60E # diagnose debug flow trace start 50
FG60E # diagnose debug enable
# Check output — should show "Denied by policy id=8"
FG60E # diagnose debug disable
