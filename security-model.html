1. Trust zones

MGMT = management plane USERS = end-user zone GUEST = internet-only untrusted zone SOC = monitoring / observability zone SERVER = protected service zone BACKUP = isolated backup domain

2. Core security principles

  • Segmentation between functional network zones
  • Least privilege access between VLANs
  • Management plane isolation
  • Explicit policy control instead of broad any-any trust
  • Guest isolation from internal services
  • Observability and troubleshooting readiness

3. Policy direction model

USERS → WAN = allow USERS → MGMT = deny USERS → SERVER = specific allow only GUEST → WAN = allow GUEST → INTERNAL = deny MGMT → ALL = admin controlled SOC → devices = monitoring flows only

4. Management plane protection

  • FortiGate admin access only from VLAN30
  • Trusted host restriction for admin account
  • Local-in deny for Guest zone
  • Switch break-glass access kept separate
