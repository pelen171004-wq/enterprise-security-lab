Overview

This lab simulates a small enterprise network security architecture built around a FortiGate NGFW and Cisco CBS250 switch. The design focuses on segmentation, inter-VLAN control, management plane protection and operational troubleshooting.

FortiGate NGFW Cisco CBS250 VLAN Segmentation Zero Trust SOC Expansion

Core Devices

FortiGate performs routing, firewall enforcement, NAT and DHCP. Cisco CBS250 provides Layer 2 switching and VLAN transport.

Current Zones

VLAN20 Users, VLAN30 Management and VLAN60 Guest are currently implemented as the core segmented security model.

Planned Zones

VLAN40 SOC, VLAN50 Servers and VLAN70 Backup extend the design toward enterprise monitoring and service isolation.

Architecture Principles

  • Centralized security enforcement through FortiGate
  • Explicit inter-VLAN traffic control
  • Guest network isolation
  • Dedicated management plane
  • Least privilege traffic model
  • Default deny mindset for east-west traffic

Traffic Flow Model

  • Users VLAN → Internet = allowed through firewall policy and NAT
  • Guest VLAN → Internet = allowed through firewall policy and NAT
  • Users VLAN → Management VLAN = denied
  • Guest VLAN → Internal networks = denied

Operational Scope

The lab is also used to validate firewall behavior through diagnose debug flow, packet sniffing, session inspection and end-to-end connectivity testing from client networks.
