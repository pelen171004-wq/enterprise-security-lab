Enterprise Security Lab - Architecture Overview

Architektonický prehľad enterprise-style security labu postaveného na FortiGate, Cisco CBS250 a segmentovanej VLAN architektúre.

Dashboard Topology Architecture

1. Cieľ labu

Cieľom tohto labu je simulovať reálnu enterprise infraštruktúru s dôrazom na:

  • network segmentation
  • management plane protection
  • inter-VLAN security policy model
  • guest isolation
  • monitoring a troubleshooting workflow
  • prechod od Network Engineer k Security Architect roli

2. Security zones

VLAN20 - USERS

User zóna s outbound internet accessom cez FortiGate NAT.

VLAN30 - MGMT

Management zóna pre správu FortiGate a ďalších zariadení.

VLAN60 - GUEST

Guest zóna s internet-only modelom a blokovaným management prístupom.

VLAN40 - SOC

Budúca monitorovacia a logovacia vrstva.

VLAN50 - SERVERS

Budúca zóna pre služby, bastion host a infra komponenty.

VLAN70 - BACKUP

Budúca backup vrstva pre PBS / NAS / recovery dizajn.

3. Trust boundaries

FortiGate tvorí hlavnú enforcement vrstvu medzi jednotlivými VLAN zónami. Každá VLAN predstavuje samostatnú trust boundary.

USER -> limited outbound access MGMT -> restricted admin access GUEST -> internet only SOC -> monitoring / logs SERVER -> protected application zone BACKUP -> isolated backup domain

4. Management plane protection

  • FortiGate management je povolený iba z VLAN30
  • Admin účet je obmedzený trusted hostom 10.10.30.10/32
  • Guest VLAN60 má local-in deny na FortiGate
  • Break-glass prístup na switch ostáva oddelený
✓ trusted host ✓ local-in protection ✓ mgmt VLAN

5. Firewall policy model

Policy model je postavený na princípe:

specific allow ↓ explicit deny ↓ default deny mindset
  • Users -> WAN = allow
  • Guest -> WAN = allow len na internet use-case
  • Guest -> internal = deny
  • Users -> Mgmt = bude explicitne kontrolované
  • Inter-VLAN access bude postupne sprísňovaný

6. Monitoring a observability roadmap

Monitoring stack ešte nie je finálne nasadený, ale architektúra s ním počíta.

  • Zabbix / observability vrstva
  • Traffic logs z FortiGate
  • Session troubleshooting workflow
  • Flow debug runbook
  • Central logging / syslog
