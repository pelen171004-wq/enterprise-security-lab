Architecture Overview

Security architecture design principles underpinning the enterprise lab. Defense-in-depth model with zero-trust segmentation, least-privilege access, and continuous monitoring.

Core Security Principles

PRINCIPLES
01
Defense in Depth
Multiple security layers — perimeter firewall, VLAN segmentation, UTM inspection, DNS filtering. No single point of failure.
Implemented: FortiGate UTM + VLAN boundaries + IPS
02
Zero Trust Segmentation
No implicit trust between zones. All inter-VLAN traffic must be explicitly permitted. Default-deny at FortiGate policy table bottom.
Implemented: 6 VLANs · Implicit deny-all policy ID 99
03
Least Privilege
Each zone has minimum required access. Guest gets internet only. Users cannot reach management plane. SOC has read-only monitoring access.
Implemented: Policy IDs 8–11 explicit deny rules
04
Management Plane Isolation
VLAN30 management traffic isolated from data plane. Switch management IP only reachable via VLAN30. Break-glass port on SW-A gi2.
Implemented: SW-A mgmt 10.10.30.2 · FG gateway 10.10.30.1
05
Visibility & Logging
All firewall policy hits logged. Security events trigger on policy violations. UTM events captured for IPS, web filter, and app control.
Implemented: FortiGate log all · Security profile logging
